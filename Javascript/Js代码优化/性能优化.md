### 内存管理
- 内存:由可读写单元组成,表示一片可操作空间
- 管理:认为的去操作一片空间的申请使用和释放
- 内存管理:开发作者主动申请空间,使用空间,释放空间
#### 为什么需要内存管理
如果我们有内存溢出，程序占用的内存会越来越大，最终引起客户端卡顿，甚至无响应。如果我们使用Node.js做后端应用，因为后端程序会长时间运行，如果有内存溢出，造成的后果会更严重，服务器内存可能会很快就消耗光，应用不能正常运行
[][http://www.dennisgo.cn/Articles/JavaScript/MemoryManagement.html]

#### 内存管理基本流程
**申请-使用-释放**
~~~javascript
// 申请
let obj = {}
// 使用
obj.name = 'zs';
// 释放
obj = null;
~~~
### 垃圾回收与常见GC算法

- javascript中内存管理是自动的
- 对象不再被引用时是垃圾
- 对象不能从根上访问到时是垃圾

#### 可达对象
- 可以访问到的对象就是可达对象(引用,作用域链)
- 可达的标准就是从根出发是否能被找到
- javascript中的根可以理解为是全局变量对象
~~~javascript
let obj = {name:'zs'};
let o = obj; //o引用{name:'zs'}
obj=null; // obj引用被断
// {name:'zs'}仍然可达
~~~
~~~javascript
function objGroup(obj1, obj2) {
    obj1.next = obj2;
    obj2.prev = obj1;
    return {
        o1: obj1,
        o2: obj2
    }
}
let obj = objGroup(
    {name: 'obj1'}, 
    {name: 'obj2'}
);
console.log(obj);
~~~

#### GC定义与作用
- GC就是垃圾回收机制的简写
- GC可以找到内存中的垃圾,并释放和回收空间

#### GC里的垃圾是什么

- 程序中不再需要使用的对象
~~~js
function func(){
    name='zs',
    return `${zs} is a coder`;
}
func()
~~~
- 程序中不再需要使用的对象
~~~js
function func(){
    const name='zs',
    return `${zs} is a coder`;
}
func()
~~~

#### 常见的GC算法

- 引用计数
  - 核心思想:设置引用数,判断当前引用数是否为0
  - 引用计数器
  - 引用关系发生改变时修改引用数字
  - 靠对象身上引用计数数值来判断是否需要被回收
  - 引用计数法优缺点
    - 优点:发现垃圾立即回收,最大限度减少程序暂停(内存不会由占满的时候)
    - 缺点:无法回收循环引用对象,时间开销大
~~~js
const user1={age:18};
const user2={age:19};
const user3={age:20};

// 被nameList应用user不会被回收
const nameList=[user1.age,user2.age,user3.age];

function fn() {
    // 未声明时挂载到window下
    const num1=1;
    const num2=2;
}
// fn执行完后num1和num2没有引用,会被GC回收
fn();
~~~
~~~js
function fn() {
    const obj1 = {};
    const obj2 = {};
    obj1.name = obj2;
    obj2.name = obj1;
    // 循环引用 无法被回收
    return 'reference count'
}
fn();
~~~
