### 循环及类型转换
#### for循环
~~~javascript
for(let i=0;i<10;i++){
    console.log(i)
}
~~~
循环步骤
- 1.声明变量 `let i= 0` 只执行一次
- 2.`if(i<10>){console.log(i)}`
- 3.i++ 可以提到循环体内
- 4.`if(i<10>){console.log(i)}`

for循环可以转化为while循环
~~~javascript
let i=0;
for(;i<10;){
    console.log(i);
    i++
}
while(i<10){
    console.log(i);
    i++
}
~~~
终止循环
~~~javascript
let i=1;
for(;i;){
    console.log(i);
    i++;
    if(i==11){
        i=0
    }
}
~~~
简单斐波那契数列
~~~javascript
// if (n === 1 || n === 2) return 1;
let n = 5;
let n1 = 1;
let n2 = 1;
for (let i = 0; i < n; i++) {
  let res = n1 + n2;
  n1 = n2;
  n2 = res;
  console.log(res)
}
~~~

break 终止循环
return 跳出循环
continue 跳出当前循环,继续剩余步骤

### 类型转化

#### 显式类型转换
**数字类型转换**
非数:
- undefined,NaN转化为NaN
- false,"",null转化为0
数字字符串:'1'=> 1,'2'=> 2,...
- Number():转化为数字
- parseInt():转化为整数,'3.14'=>3
  - 第二个参数表示进制,可以将部分字母转化为数字
  - 16进制 0123456789abcdef
  - parseInt('b',16)=>11
  - parseInt转换,数字开头转数字,非数字开头转NaN,'123abc'=>123,'abc123'=>123
- parseFloat():转化为浮点数
  - '3.14'=>3.14
**布尔类型转换**
- Boolean():转化为布尔类型
  - undefined,null,NaN,"",0,false转化为false,其他都转化为true
**字符串类型转换**
- String()
- xxx.toString 10.toString=>报错,10..toString()=>10,10.1.toString()=>10.1

#### 隐式类型转换
- +'123'=> 123,-'123'=>123 
- *,/,-,% 做运算时都要做隐式转换,string转化为number类型
- >,<,>=,<= 做运算时也会进行隐式转换,首先转换number,其次会转化为ascii值
- == 做运算时也会进行隐式转换
- === 做运算时不进行隐式转换,会进行数据类型检查
- NaN 不等于任何东西
- undefined == 0 false
- null == 0 false
- undefined == null true
~~~javascript
let a='123';
a++;
console.log(a);//124
~~~
~~~javascript
let a = 'a' + 1;
let b = '2' * 2;
console.log(a); // a1
console.log(b); // 4
~~~
~~~javascript
let a = 1 > '2';
let b = '2' > 1;
console.log(a); // false 
console.log(b); // true 
~~~
~~~javascript
let a = 2 > 1 > 3;
let b = 2 > 1 == 1;
let c = 3 > 2 == 1;
let d = 3 > 2 == 2;
console.log(a); // false 2>1=>true=>1<3=>false 
console.log(b); // true 2>1=>true=>1==1=>true
console.log(c); // false 2>1=>true=>1!==1=>true
console.log(d); // false 2>1=>true=>1!==2=>false
~~~
#### isNaN()

console.log(isNaN('123')); // false

### 函数
高内聚,低耦合 模块的单一责任制问题
- 耦合:各个代码块间重复的代码
- 高内聚:开发的模块相关性强,模块内部独立完成任务
- 低耦合:重复代码抽离出来组成单独的模块,完成特定功能
某一块代码独立完成该任务
用函数可以很有效的解耦

#### 函数声明
- function 声明函数的关键字
- 表达式 + 字面量 `const test=function test1() {}` 外部不可调用test1(),但是在函数内部可以进行调用test1() 
- 匿名函数表达式(函数字面量) `const test=function(){};`
~~~ javascript
function test(){
  var a = 1,b = 1;
  var c = d = 1;// => d=1; var c=d d没有声明,挂到全局对象上
  console.log(a,b,c,d); // 1,1,1,1
}
test();
console.log(a); // a is not defined
console.log(b); // b is not defined
console.log(c); // b is not defined
console.log(d); // 1
~~~
#### 函数命名规则
- 不能数字开头,可以包含数字
- 小驼峰命名法

#### 形参和实参
- 形参就是变量,形式占位参数=>形式上的参数
- 实参就是实际参数,给变量赋上实际的值
- 赋值时参数一一对应
- 形参个数大于传入的实参,函数不会报错,没有传入的按undefined处理
- 实参个数大于形参,函数也不会报错,多传入的实参忽略
- 形参和实参的个数可以不相等
- 实参在函数内部可以用`arguments接收`
- 在函数内部可以修改实参的值
- 形参的第一个参数和arguments[0]不是一个东西 a=3是存在栈内存内,arguments[0]存在在堆内村中,arguments这个伪数组在队中

~~~javascript
function test(a,b){
  a = 3;
  b = 3;
  console.log(arguments[0]);
  // a和arguments[0]不是一个东西 a=3是存在栈内存内,arguments[0]存在在堆内村中,arguments这个1伪数组在队中
  console.log(arguments[1]);
}
test(1,2); // 3,3
test(1); // 3 undefined
~~~
~~~javascript
function sum() {
  let res = 0;
  for (let i = 0; i < arguments.length; i++) {
    res += arguments[i]
  }
  return res;
}
console.log(sum(1, 2, 3, 4, 5, 6)); // 21
~~~
